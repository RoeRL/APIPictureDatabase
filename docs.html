PictureStorage API

PictureStorage API adalah sistem penyimpanan gambar yang dibangun menggunakan .NET 10 dan SQLite. Mendukung pengoperasian data gambar mulai dari proses upload, delete, dan retrieval melalui API.

Base URL

http://api.roearchive.site/api

Endpoints

POST /Pictures

Upload Gambar: Upload gambar baru ke server, disimpan pada volume lalu menyimpan catatan metadata ke dalam database.

Proses ini akan melakukan generate Unique Identifier (UUID) untuk mencegah konflik antar nama file, memberikan keamanan pada nama asli file, dan logging ke dalam database.

Request Parameters (Body / form-data)

KeyTypeDescriptionRequired
fileFileImage file yang akan di upload (.jpg, .jpeg, .png, .gif, .bmp, .webp).Yes

Success Response

Code: 200 OK
Returns ID gambar atau pesan success confirmation

GET /Pictures

Get All Pictures (Paginated): Mengambil daftar metadata semua gambar yang diunggah, diurutkan berdasarkan tanggal pembuatan.

Untuk mencegah sistem overloading pada server atau client, dibuatlah Server-Side Pagination. Dengan cara memberikan query parameters, maka server akan melakukan kalkulasi bagian mana yang akan dilakukan fetch.

URL Query Parameters

ParameterTypeDefaultDescription
pageNumberInteger1Halaman hasil yang akan di retrieve.
pageSizeInteger10Jumlah image per halaman.

Success Response Example

Code: 200 OK

[
  {
    "id": "29f4aead-9291-4fa7-bf46-77a15fe51938",
    "fileName": "cat-funny.png",
    "originalFileName": "cat-funny.png",
    "filePath": "/app/data/pictures/29f4aead.png",
    "created": "2026-03-26T11:48:24Z"
  }
]

GET /Pictures/view/{id}

View a Single Picture: Stream gambar secara langsung pada client.

Tidak memberikan data JSON, endpoint ini akan mencari file secara langsung dan memberikan return gambar sebenarnya. Gunakan ini secara langsung <img src="..."> pada bagian frontend.

URL Parameters

ParameterTypeDescriptionRequired
idStringUnique ID (UUID) dari gambar.Yes

Success Response

Code: 200 OK
Returns gambar secara langsung (e.g., image/jpeg atau image/png).

DELETE /Pictures/{id}

Delete a Picture: Menghapus gambar dari database dan file secara permanen.

Akan membersihkan dengan dua langkah: mencari dan menghapus file secara langsung pada storage untuk mengosongkan storage, lalu menghapus metadata pada database SQLite.

URL Parameters

ParameterTypeDescriptionRequired
idStringUnique ID (UUID) dari gambar.Yes

Success Response

Code: 200 OK or 204 No Content

Storage Quotas

Maximum Capacity: 25 GB

Jika storage sudah menyentuh 25GB, maka tidak dapat melakukan upload image baru.

  • Jika upload melewati limit, server akan menolak file dan memberikan return 413 Payload Too Large.

Standard Status Codes

  • 200 OK / 204 No Content - Request successful.
  • 400 Bad Request - Request invalid (e.g., empty file upload).
  • 404 Not Found - Requested resource tidak tersedia.
  • 413 Payload Too Large - 25GB storage kuota sudah penuh.
  • 500 Internal Server Error - Server-side exception error.
